# Log Viewer

Простой локальный просмотрщик логов партий.

## Запуск

```bash
cd log-viewer
node server.js
```

По умолчанию — `http://localhost:5174`.

Если порт занят:

```bash
PORT=5180 node server.js
```

## Формат логов

Backend пишет КАЖДУЮ партию в отдельную директорию:

```
logs/<mode>/<red>-<blue>-<localTs>/
  meta.json        # Полная метаинформация: mode, players, IP, config, итог, длительность
  game.log.jsonl   # Упрощённые «игровые» события — это и проигрывает просмотрщик
  aux.log.jsonl    # (опц.) Сервисный лог: телеметрия бота, фейлы — просмотрщик игнорирует
```

Скопируй такую папку в [`log-viewer/logs/`](logs/) и нажми «Обновить» — она
появится в списке.

### Словарь игровых событий (одинаковый для PvP и solo)

| kind             | смысл                                                       |
|------------------|-------------------------------------------------------------|
| `setup_mine`     | расстановка стартовой мины (или её снятие при toggle)       |
| `setup_confirmed`| игрок подтвердил расстановку                                |
| `game_started`   | обе расстановки подтверждены, фаза 1 началась               |
| `zone_select`    | выбор зоны 3×3                                              |
| `cell_open`      | захват одной клетки (аккорд → N таких событий подряд)       |
| `mine_hit`       | взрыв мины (жизнь -1, мина удаляется)                       |
| `mine_defused`   | разминирование (клетка переходит к игроку)                  |
| `phase3_mine`    | игрок поставил мину в фазе 3                                |
| `turn_end`       | конец хода → ход переходит сопернику                        |
| `game_finished`  | партия завершилась (winner + reason)                        |

Флаги и аккорды как отдельные сущности **не пишутся**. Аккорд раскладывается
в последовательность `cell_open` с `viaChord:true`.

## Управление

В списке кликни на партию — откроется поле.

В режиме просмотра:
- **← / →** — шаг назад / вперёд по событию.
- **Home / End** — в начало / в конец партии.
- **Esc** — назад к списку.
- Поле **полностью открыто**: видно мины обоих игроков, цифры (только внутри
  активной display 3×3), владельца каждой клетки и текущую расстановку.

## API

- `GET /api/sessions` — список сессий, считанный из `meta.json` каждой
  директории.
- `GET /api/session/<id>` — `{ meta, events }` для конкретной партии.

## Дефолты конфига

Если в meta.json почему-то нет `config`, подставляются:

- `boardSize`: 10
- `initialMinesRed`: 7
- `initialMinesBlue`: 9
- `maxLives`: 3
