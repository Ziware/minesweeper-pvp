# Log Viewer

Простой локальный просмотрщик логов партий.

## Запуск

```bash
cd log-viewer
node server.js
```

По умолчанию — `http://localhost:5174`.

Если порт занят:

```bash
PORT=5180 node server.js
```

## Как добавить лог

Положи файл `.log.jsonl` (или папку с `game.log.jsonl` внутри) в каталог
[`log-viewer/logs/`](logs/). Просмотрщик подхватит файлы автоматически —
просто обнови список в браузере (или нажми «Обновить» в шапке).

Принимаются:
- одиночные файлы `*.jsonl` / `*.log.jsonl` — каждая строка = одно JSON-событие;
- директории формата серверного `gameLogger` (с файлом `game.log.jsonl` внутри);
- директории с произвольными `*.jsonl` файлами.

## Управление просмотром

Из главного списка кликни на партию — откроется поле.

В режиме просмотра:
- **← / →** — шаг назад / вперёд по ходам.
- **Home / End** — в начало / в конец партии.
- **Esc** — назад к списку.
- Поле **полностью открыто**: видно мины обоих игроков, цифры, владельца
  каждой клетки и текущую расстановку (метки, фазу, чей ход).

## Что показывается на каждом шаге

- Полная доска со всеми минами (красные и синие).
- Открытые числа на собственных клетках каждого игрока (вычисляются по
  правилам игры — соседние **вражеские** мины в радиусе 1).
- Подсветка активных зон: `displayZone` (3×3) и `actionZone` (5×5).
- Жизни, фаза, чей ход.
- Лог-полоска с описанием последнего события («red captured (4, 5)»,
  «blue defused (3, 2) — mine», «chord finished — 3 captures» и т. п.).

## Какая логика реализована

Просмотрщик — **сам себе движок**. Он берёт лог-события и
последовательно проигрывает их над начальным состоянием:

1. `room_created` / `player_joined` — настройка игроков.
2. `setup_mine_toggled` — расстановка стартовых мин.
3. `setup_confirmed` / `game_started` — переход к фазе игры.
4. `zone_selected` — выбор зоны (показ 3×3 и 5×5).
5. `cell_captured` — захват клетки (раскраска).
6. `mine_exploded` — клетка взрывается, жизнь -1.
7. `cell_defused` — разминирование (с миной или без).
8. `chord_started` / `chord_finished` — аккорд.
9. `phase2_ended` / `phase3_mine_placed` / `phase3_ended` —
   фаза 3 и переход хода.
10. `mark_toggled` — флажки/вопросики (показываются как метка).
11. `game_finished` — финиш.

Логика преднамеренно «толерантная» к недостающим / неизвестным событиям —
если в логе появится новое поле, просмотрщик его проигнорирует и
покажет остальное.

## Где взять конфиг

Конфиг (`boardSize`, `initialMinesRed`, и т. д.) берётся из ивента
`game_started` или `room_created`, либо подставляются дефолты:

- `boardSize`: 10
- `initialMinesRed`: 7
- `initialMinesBlue`: 9
- `maxLives`: 3
