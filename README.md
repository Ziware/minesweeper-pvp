# Minesweeper PvP

Многопользовательская дуэльная вариация классического сапёра для двух игроков (красный и синий). Каждый владеет своей половиной поля 10×10, расставляет мины и пытается прорваться к штабу противника, разведывая местность подсказками-цифрами.

🔗 Адрес: [minesweeper-pvp.ru](http://minesweeper-pvp.ru/)

📐 Внутреннее устройство проекта описано в [`ARCHITECTURE.md`](ARCHITECTURE.md).

## 🎯 Цель

Главная цель — захватить штаб противника: зону 1×2 клетки по центру края доски. Захват любой клетки штаба сразу приносит победу. Если у одного из игроков закончится время на партию — он проигрывает.

Партия играется с шахматными часами: **X минут на всю партию + Y секунд за каждый завершённый ход**. Часы запускаются после фазы расстановки мин и идут на протяжении всех трёх фаз хода вплоть до передачи хода противнику. Часы не идут на этапе расстановки первых мин.

## 🗺️ Поле

Поле 10×10. Красный владеет верхней половиной, синий — нижней. Штаб красного находится по центру верхнего края, штаб синего — по центру нижнего края. Штаб нельзя заминировать. В начале каждый расставляет по **8 мин** на своей половине. Мины противника скрыты. Ходить начинает красный.

## 🔄 Ход игрока (3 фазы)

### Фаза 1 — Выбор зоны
Кликните на любую клетку поля — вокруг неё появится зона **3×3** (жёлтая рамка) и зона **5×5** (синяя рамка). В зоне 3×3 должна быть хотя бы одна *доступная* союзная клетка: такая клетка соединена с вашим штабом непрерывным путём по соседним союзным клеткам через общие стороны. На своих клетках в зоне 3×3 появятся цифры — сколько мин противника находится рядом с каждой клеткой.

### Фаза 2 — Захват клеток
В пределах зоны **5×5** можно захватывать клетки противника кликом по ним. Захват возможен только если захватываемая клетка соседствует по общей стороне хотя бы с одной *доступной* клеткой вашей территории — диагонали не считаются.

Если в клетке **есть мина** — вы теряете жизнь, мина исчезает, клетка остаётся за противником, ход переходит к фазе 3.

Доступную для захвата клетку можно **разминировать** (`Ctrl+Click`) в пределах лимита разминирований на этот ход. В начале игры лимит — **1 разминирование на ход** для каждого игрока. После каждых **5 совместных ходов** (после 5-го, 10-го, 15-го хода и т. д.) лимит увеличивается на **+1 для обоих игроков**. Неиспользованные разминирования сгорают в конце хода — накопить их нельзя.

Если в разминируемой клетке есть мина — она убирается, клетка переходит к вам, ход продолжается. Если мины нет — клетка переходит к вам, фаза 2 завершается и начинается фаза 3.

Можно добровольно завершить фазу кнопкой *«Завершить захват»*.

### Фаза 3 — Расстановка мин
Можно поставить от **0 до 3 мин** на свободные доступные клетки своей территории. Если выбранная в фазе 1 зона **5×5** содержала хотя бы одну клетку *вашего* штаба — лимит увеличивается до **4 мин** (+1 за защитную зону). Завершить расстановку можно вручную; поражения из-за нехватки места для мин нет.

## 🏁 Завершение игры

- Мгновенная победа за захват любой клетки штаба противника.
- Победа противника, если потеряны все **3 жизни** (взрывы на минах).
- Поражение у того, чьё время на партию первым истечёт (шахматные часы).

## 🔢 Цифры

Цифра на клетке показывает сколько мин противника находится в 8 соседних клетках. **0** — мин рядом нет. Мины на своих клетках в зоне 3×3 скрыты — видны только цифры. Захват при этом выполняется только через 4 клетки по общей стороне.

## ⚖️ Баланс игры

Все ключевые числовые параметры (размер поля, количество мин, лимиты разминирований, бонус мин за штаб в зоне действия, пресеты шахматных часов и т.д.) централизованы в [`packages/shared/src/balance.json`](packages/shared/src/balance.json). После изменения значений и пересборки они автоматически применяются и в логике, и в текстах подсказок UI.

| Параметр | Путь в JSON | Значение по умолчанию |
| --- | --- | --- |
| Размер поля | `board.size` | 10 |
| Стартовых мин у игрока | `board.initialMines` | 8 |
| Жизней у игрока | `player.maxLives` | 3 |
| Мин за ход в фазе 3 | `phase3.minesPerTurn` | 3 |
| Бонус мин за штаб в зоне 5×5 | `phase3.hqInActionZoneBonusMines` | 1 |
| Стартовых разминирований / ход | `defuse.initialPerTurn` | 1 |
| Каждые N ходов +1 разминирование | `defuse.grantInterval` | 5 |
| Пресеты шахматных часов | `timeControls.presets` | 2+3, 3+5, 5+7 |
| Индекс пресета по умолчанию | `timeControls.defaultPresetIndex` | 1 (3+5) |

## 🛠️ Технологии

- **Backend:** Node.js, TypeScript, Express, Socket.IO
- **Frontend:** React 18, TypeScript, Vite, CSS Modules
- **Shared:** TypeScript-пакет с общими типами и геометрическими помощниками
- **Audio:** Web Audio API + 8 предзагруженных `.wav`-эффектов
- **Хранение настроек:** `localStorage`
- **Деплой:** Docker + docker-compose

## 🚀 Запуск локально

```bash
yarn install
yarn build
yarn workspace @minesweeper-pvp/backend start &
yarn workspace @minesweeper-pvp/frontend dev
```

Откройте [http://localhost:3000](http://localhost:3000) в двух вкладках/браузерах: один игрок создаёт комнату, второй вводит её код.

## 🐳 Деплой через Docker

```bash
./deploy.sh
```

Скрипт пересобирает образы и поднимает frontend (nginx) + backend (Node) через `docker-compose`.
